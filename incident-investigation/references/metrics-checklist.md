# Metrics Checklist

排查线上故障时，优先检查以下指标。

## 应用层

- 5xx 错误率
- 请求量 QPS
- p95 / p99 延迟
- 超时数

## 资源层

- CPU 使用率
- 内存使用率
- 磁盘 IO
- 网络连接异常数

## 依赖层

- 数据库连接超时数
- Redis 超时数
- 下游 HTTP 依赖错误率
- MQ 堆积情况

## 判断提示

- 如果错误率升高但 CPU / 内存正常，优先怀疑外部依赖或网络访问异常
- 如果延迟升高且超时数上升，重点看下游连接、线程池、数据库响应
- 如果最近刚有变更，优先把变更时间和异常时间做对齐
